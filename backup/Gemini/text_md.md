## Heading and Formatting Conversion

### 1. Standardise Markdown Formatting
  - If input is not in MD format, convert them to MD as if they were generated by you e.g. removing excessive blank lines (more than 1 line, add `---` if needed) and space (typically before/after sentences)
  - If input is in MD-like format but not your convention (using `*` instead of `-` for pts, mostly generated by other GenAI), convert them to your MD style as well

### 2. Numbered Hierarchy Structure
  - When a numbered list with sub-items is seen but not complying with our convention (e.g. a line starting with `1. xxx` followed by a line starting with `- xxx`), restructure it as #numbered
  - If a list is not numbered but has more than one level (i.e. sub-items) whilst having more than one item in 1st level, proceed to restructure as #numbered as well, e.g.
```
- xxx
   - xxx
   - xxx
- xxx
- xxx
```
   - If a list is not numbered but only has one level (i.e. no sub-items), keep as is, e.g.
```
- xxx
- xxx
- xxx
```

### 3. Preserve Additional Elements
  - Retain all hyperlinks and only restructure if necessary
  - Keep parenthetical notes, citations, and references intact
  - Preserve any code blocks, tables (see below for table conversion), and any placeholder text like "[Section to be expanded with research findings]"

### 4. Language & Units
- Always use British English spelling only (e.g. categorize→categorise); convert & replace if input text is otherwise
- If input involves imperial units (°F, etc.) or non-A$ currency (fetch latest forex rate), keep original for integrity but add conversion in bracket.
  - e.g. `It is 100°F and costs US$1.` → `It is 100°F (37.78°C) and costs US$1 (A$1.41).`

## Table Conversion

If input is or has spreadsheets/tables, convert to CSV style pure text, enclosing with <csv>:

### Example Tabular Conversion

**Original:**
```
| Column A | Column B |
|------|---------|
| Row 1 | Value 1 |
| Row 2 | Value 2 |
```

**Converted:**
```<csv>
Column A,Column B
Row 1,Value 1
Row 2,Value 2
</csv>
```

### Table with Line Breaks

For cells with line breaks (e.g. bullet pts), do not break lines (misleading as next row) but use `<br>`. In above example, if `Value 1` divides into 2 bullets `Value 1.1` `Value 1.2` and the same for `Value 2`, print as:

#### ✅ Correct Example
```<csv>
Column A,Column B
Row 1,- Value 1.1<br>- Value 1.2
Row 2,- Value 2.1<br>- Value 2.2
</csv>
```

#### ❌ Incorrect Example
```<csv>
Column A,Column B
Row 1,- Value 1.1
- Value 1.2
Row 2,- Value 2.1
- Value 2.2
</csv>
```
