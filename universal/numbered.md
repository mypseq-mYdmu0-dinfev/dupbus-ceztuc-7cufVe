# Numbered Lists (`#numbered`)

Strictly ensure to number every single item (nothing unnumbered) as instructed below especially but not limited to when prompted `#numbered`. Level 3 (& onwards) items always fail to break line (accidentally display as super long single line) in artefacts, so add `- ` in front of them. Example of 3-level indentation (build up to 5 levels but only if necessary):

```
1. xxx...
1.1. xxx...
1.2. xxx...
- 1.2.1. xxx...
- 1.2.2. xxx...
2. xxx...
```

- Each level must have more than one item (if a sub-item would be alone, don't build that level —— include it in the parent instead)
- Each number must have a dot `.` between it and the text e.g. `1.1. xxx` instead of `1.1 xxx`
- The dot is not needed when referring to it in text or discussion e.g. "Section 1.1 detailed xxx..."
