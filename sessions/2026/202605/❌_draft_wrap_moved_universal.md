# Monthly Wrap Template (Destination: `universal/wrap.md`)

*Triggered by: `#wrap` | Current_TS: `TZ='Australia/Sydney' date +"%Y%m%d%H%M"`*

---

*Close with 1 file*

Filename: `wrap_[TS].md`
Location: `/sessions/[YYYY]/[YYYYMM]/` —— one per month

---

### Template

```markdown
# Monthly Wrap: [YYYYMM]
*[Heading max 8w]*

## Sessions
1. `[YYYYMM]/close_[TS].md` —— [max. 8w descr]
2. `[YYYYMM]/[CP_folder]_close_[TS].md` —— [ditto]
[... all close_ files of the month, in chronological order]

## Key Themes
[Cross-session patterns; recurring decisions; ongoing work threads]

## Key Decisions
[Decisions from this month worth carrying into next month]

## Carried Forward
[Open items from close_ files not yet resolved; tagged 🔴/🟡/🟢]

## Notable Files
[Non-sessions files created or meaningfully modified this month; enclosing_folder/filename]

## For Next Month
[Standing context that next month's sessions should know; ONLY what's not in sections above; optional]
```

---

### Rules

- Read ALL `close_` files in the target month's subfolder before writing
- Include ALL close_ files (both CP-prefixed and non-prefixed) in §Sessions
- Synthesise —— don't just copy; the wrap_ should be faster to read than all close_ files combined
- One wrap_ per month; if one already exists, update rather than create a new one
