# Response to query_202606050319

1. What the problem was:
- 1.1. Finder's **"Date Added"** is the filesystem **catalog** attribute `ATTR_CMN_ADDEDTIME` (Swift: `addedToDirectoryDate`) —— a different field from the Spotlight metadata that `xattr`/`mdls` expose.
- 1.2. My first two attempts only wrote the `com.apple.metadata:kMDItemDateAdded` extended attribute, which is merely Spotlight's mirror; Finder's column reads the catalog field, so it never moved. Swift's `addedToDirectoryDate` is get-only, so it couldn't set it either —— which is why "nothing changed".
2. How I fixed it:
- 2.1. Called the low-level `setattrlist()` syscall (via Python `ctypes`) with `ATTR_CMN_ADDEDTIME = 0x10000000` and a `struct timespec` of the target time, then refreshed the Spotlight xattr so both agree. Confirmed via Foundation readback: **Date Added = 04/06/2026 19:58 AEST**.
3. Reusable script created & tested: `Culous Codes/DAMF.py` (Date Added Modify Fixer).
- 3.1. Reads exactly one `.txt` beside itself (excludes `temp.txt`; stops if 0 or `~`2⁺ eligible) —— Line 1 = filename, Line 2 = `YYYYMMDDHHmm` (Sydney time, DST-aware).
- 3.2. Searches the whole repo for that filename; stops if 0 or `~`2⁺ matches; otherwise sets Date Added via the same `setattrlist()` method.
- 3.3. End-to-end test passed using your `202606050319.txt` (set 19:58 as you specified); the multiple-`.txt` and `temp.txt`-exclusion guards both fired correctly during testing.
