# Response to query_202606201812.md

1. The precise mandate, ready to embed in a file header or CLAUDE.md rule:

> **Read/append-only.** Use the `Read` tool to read this file. To write, use ONLY a Bash `>>` (append redirect), e.g. `echo "..." >> [path]` or `printf "..." >> [path]`. NEVER use the `Edit` or `Write` tools on this file — both overwrite and will corrupt concurrent entries.

2. Why each constraint:
- 2.1. `Read` tool — safe for any number of concurrent readers; no lock needed
- 2.2. `echo/printf >>` — the OS guarantees atomic write for payloads below PIPE_BUF (`~`4 KB on macOS); concurrent single-line appends will not interleave
- 2.3. `Edit` banned — edits in place by rewriting a region; a concurrent write between read and write clobbers the other session's content
- 2.4. `Write` banned — overwrites the entire file; any concurrent content is lost

3. If any single append will exceed `~`4 KB, add a note to write to a temp file first then `cat tmp >> [path] && rm tmp` to keep each OS write atomic.
