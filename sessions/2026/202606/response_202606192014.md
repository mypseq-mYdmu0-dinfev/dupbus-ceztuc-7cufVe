# Response to "so if i wanna schedule a task in an existing session, i needa ask you to do it for me rather than myself w/ a certain command?"

1. To **create** the schedule: yes, you ask me (e.g. type `/schedule` or just describe it) and I call the tool — there's no user-facing command that bypasses me.

2. But the core constraint isn't about who creates it — it's that **a scheduled task always spawns a new session**; it cannot resume or inject into an existing one, full stop.

3. So "scheduling a task in an existing session" isn't a supported concept — the closest you get is:
- 3.1. Schedule the task (fires in a new session)
- 3.2. Manually bring its output back into the session you care about (e.g. read the file it wrote, or copy-paste)
