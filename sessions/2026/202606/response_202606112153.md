5. Temporary public sharing —— how it actually works on GitHub
- 5.1. GitHub has **no native expiring/temporary links**. The realistic model for "anyone with a link, no password" is exactly your fallback: make it public → share → make private or delete when done.
- 5.2. **Public repo:** `Settings → General → Danger Zone → Change visibility → Public`. Then the URL `github.com/Culous/<repo>` (and any file/folder URL under it) is viewable by anyone, no login, no pw. To stop sharing: flip back to **Private**, or **Delete** the repo. (Caveat: while public it's also crawlable/searchable; that's the trade-off for no-login access.)
- 5.3. **Single files / snippets → secret Gist:** `gist.github.com` → paste content → **Create secret gist**. Gives an unlisted, hard-to-guess URL anyone can open without login; delete anytime. "Secret" = unlisted, not access-controlled (anyone with the link sees it) —— which is exactly what you want for temporary public sharing.
- 5.4. **Private repo sharing does NOT fit your goal:** sharing a private repo requires inviting a collaborator (needs *their* GitHub account) —— no anonymous link. So for "anyone with a link, no pw", it must be **Public repo** or **secret Gist**.
