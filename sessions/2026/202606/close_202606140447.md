## 2. Decisions
- 2.1. `dlint.py` is RUN, never READ (documented in a `cscpt/README.md`); its lists can grow without costing context tokens.
- 2.2. RED = zero-tolerance, fires only on genuine breaches (em dash, exact Americanisms, mid-sentence colon, comma immediately inside a closing quote). YELLOW = conditional, may remain if justified in `response_`.
- 2.3. en dash = YELLOW (not RED) to avoid forcing conditional-acceptance on valid ranges; `-ize` generalisation = YELLOW with exceptions; exact Americanisms stay RED.
- 2.4. Hart's quotation —— flag ONLY the char immediately before a closing quote (comma → RED, lone period → YELLOW, ellipsis exempt).
- 2.5. `--quick` (British spelling + Hart + #numbered) is the all-output/comms variant; it NEVER rewrites and MASKS code (fenced blocks + inline spans) so real code never false-flags. Embedded deliverables are instead caught by extracting them and running FULL `--text` (per `writing.md`).
- 2.6. Hook fires ONLY on `response_*.md` (never code/automations/`/seek/`); `dlint_hook.sh` is a bash fast-path so Python spawns only when `response_` is involved. Token cost is zero unless it blocks.
- 2.7. Deliverable FULL-lint cannot be hook-enforced (no deterministic filename pattern); relies on §3.7.3 + `writing.md` + the user's `#writing`.
