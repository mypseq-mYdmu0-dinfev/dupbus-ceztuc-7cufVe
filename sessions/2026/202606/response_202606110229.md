# Response to query_202606110229
*MBA M1 setup for CAI + CC*

1. **Core setup** —— yes, broadly correct: clone repo via GH Desktop, install CAI, open CC from it. That gets the session running. A few layers of detail below though.

2. **The FURY 2TB problem** —— this is the biggest practical issue. The repo currently lives at `/Volumes/FURY 2TB/...`, and CLAUDE.md hardcodes two more FURY paths:
- 2.1. Downloads: `§8.8.1` → `/Volumes/FURY 2TB/Fury Downloads`
- 2.2. Screenshots: `§8.8.2` → `/Volumes/FURY 2TB/Fury Pictures/Screenshots`
- 2.3. On MBA, the repo clone will sit on the internal drive (e.g. `~/Documents/GitHub/dupbus-ceztuc-7cufVe`), so all **repo-internal** paths resolve fine —— but §8.8.1 & §8.8.2 will silently mis-resolve unless FURY is plugged into MBA too, or those paths are updated for MBA. Suggest: either accept FURY as always-attached, or keep the MBA-specific overrides noted somewhere (e.g. a short note in CLAUDE.md or a separate MBA variant)

3. **CC memory is machine-local** —— the auto-memory at `~/.claude/projects/*/memory/` lives on the Mac Mini only. MBA starts with a blank slate. This IS perceivable: I won't remember anything (your email, current date override, feedback patterns) until you run a session or two and memory rebuilds. Easy mitigation: manually copy `~/.claude/projects/*/memory/` from Mac Mini to MBA once after setup.

4. **MCP servers need reconfiguration** —— project-level `.claude/settings.json` syncs via GH (so tool permissions, hooks carry over). But global `~/.claude/` settings and any OAuth/auth tokens (Gmail MCP, Calendar MCP, Chrome extension, etc.) are machine-local and must be re-authorised on MBA. Each MCP that needs a browser login will prompt you on first use.

5. **Global `~/.claude/` settings** —— things like your account login, model preferences set globally, and any user-level settings don't travel with the repo. These are quick to redo but worth noting.

6. **Simultaneous use risk** —— if both machines run CC on the same repo concurrently without disciplined git pull/push habits, you'll get merge conflicts on session files. Low risk if you treat it as "only one active machine at a time", which seems to be your intent.

7. **Perceivable differences**
- 7.1. First few sessions: no memory → I'll lack context I normally have (your email, feedback rules, etc.)
- 7.2. Path-broken tools: screenshots & downloads point to FURY; either plug FURY in or update paths
- 7.3. MCP reconnection prompts on first use of each connector
- 7.4. Otherwise: workflow, responses, session files, CLAUDE.md behaviour —— identical

8. **CAI + inference performance** —— yes, all Claude computation is serverside. M1 MBA is irrelevant for AI quality or speed. Local shell commands (bash, file I/O) run on MBA hardware, but M1 is more than fast enough for any file/git ops in this workflow —— you won't notice a difference.

9. **Recommended order of ops**
- 9.1. Install GH Desktop → clone repo to MBA internal drive
- 9.2. Install CAI → sign in → open CC
- 9.3. Copy `~/.claude/projects/*/memory/` from Mac Mini to MBA (preserves memory)
- 9.4. Optionally copy `~/.claude/settings.json` from Mac Mini (preserves global prefs)
- 9.5. On first session: re-authorise each MCP connector as prompted
- 9.6. Update or conditionally handle §8.8.1 & §8.8.2 paths if FURY won't be plugged into MBA
