7. Why an edit shifts BOTH Date Modified AND Date Added (you asked)
- 7.1. It's not a macOS/GH bug —— it's atomic/"safe" saving. Editors (and CC's Edit/Write tool) save by writing a temp file then renaming it over the original. On APFS, that rename is a NEW directory entry, so "Date Added" (= when the entry joined the folder) resets to now, and "Date Modified" updates too. Editing in place would avoid it, but most editors don't, and it's not worth disabling. `DATS.py` is the clean corrective.
