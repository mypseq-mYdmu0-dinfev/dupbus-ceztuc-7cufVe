48. One optimisation genuinely worth raising —— the cost driver is compounding full-board RE-READS:
- 48.1. In test 4 the four SAs burned `~`360–385k tokens for a 15k board. The main reason: every cycle, each debater AND the Observer re-reads the WHOLE board (the `Read` tool on the full file), which grows each turn —— so each agent's context accumulates many copies of an ever-larger board, and cost compounds roughly quadratically with debate length. The "light" Observer wasn't light (93k) for exactly this reason.
- 48.2. Fix —— DELTA reads: each agent tracks the last line it has seen and reads only the NEW lines since (the `Read` tool's `offset`, or Bash `tail -n +[N]`). It still sees everything fresh (no stale reads; `THE_END` still caught), but never re-pays for old content —— its earlier reads already sit in its context. This would cut per-debate cost substantially (the Observer most of all). It's the single highest-value change left.
