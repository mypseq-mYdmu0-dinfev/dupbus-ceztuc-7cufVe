## Issue 2 —— Font & Layout (dual mode built)

46. The other discrepancy —— **fonts changing, hence layout** —— is real and expected (yes, I flagged it earlier). The docs use fonts LibreOffice doesn't ship: `Aptos` (the new Office default), `Georgia`, `Fira Mono`, `Play`, `Arial Unicode MS`, `Noto Sans Symbols`; only `Calibri` has a metric-compatible LibreOffice substitute (`Carlito`). LibreOffice swaps each missing font for its nearest match, and the differing glyph widths shift line-breaks, spacing, pagination, and table widths. That is the inherent Quick-Mode engine-difference cost —— not fixable inside LibreOffice.

47. So I built your **dual mode** (executed —— no QB; MS-only, iWork untouched):
- 47.1. **Quick Mode (default):** drop an MS file → LibreOffice converts it silently into the run's `[TS]/` subfolder, original deleted on success. Exactly as now.
- 47.2. **Full Mode (auto, on re-run):** if that file's PDF already exists in ANY earlier `[TS]/` subfolder (i.e. you re-dropped it because Quick wasn't good enough), the run **immediately pops up "⚠️ Full Mode: manual intervention needed"** once, up front —— then re-converts via the native Office app (best fidelity); you grant file access when prompted. Output → the current run's `[TS]/`, original deleted, normal completion pop-up.
- 47.3. The decision is per file, by base name (`X.docx` ↔ a prior `X.pdf`), so one run can mix Quick (new files) and Full (re-dropped ones); the alert fires once if ANY file needs Full.

48. Design calls I made (flag if you'd prefer otherwise —— none blocking):
- 48.1. **Same-minute guard, made general:** if a `[TS]/` folder for the current minute already exists, the run stops and alerts rather than reusing/overwriting it. You raised this for the Full-Mode re-run; I applied it to every run because the overwrite risk is identical.
- 48.2. The end pop-up is now honest (last turn's fix): a success message with a count only when files actually converted; otherwise a warning, and it clears the empty folder. (Your earlier no-op run left an empty `202606181946/` —— harmless; delete at leisure, future no-op runs self-clean.)

49. **Important caveat —— Full Mode will NOT make `Company - Background` portrait.** Word honours the same landscape page setup, so its PDF is landscape too. Full Mode fixes the **font/layout** fidelity, not the (correct) orientation —— for portrait you must edit the source (45).
