## 1. Coverage
- 1.1. Diagnosed the CWI/OTGC stale-fetch ("v01") bug —— two URL-keyed cache layers (Claude web-fetch ~15 min + GitHub raw CDN).
- 1.2. Designed & built the `#sync` mechanism: pins every OTG file URL to its last-commit SHA permalink (immutable → never stale), and pins each index's own permalink inside its prefs file.
- 1.3. Built the toolset, wired the harness (permission rule + Date-Added hooks + git guard), and stress-tested across `universal`, `career`, `dissertation`.
- 1.4. Fixed a Date-Added side-effect on `google.md` and built a permanent Date-Added preservation hook.

## 2. Decisions
- 2.1. SHA permalink (per-file last-commit SHA) chosen over query-param cache-bust (CWI ignores query strings) and over the old versioned-duplicate-file scheme (repo clutter).
- 2.2. Workflow: the USER commits content + control-file edits in their own commits; `#sync` commits ONLY the 2 control files; it ABORTS untouched if the index, prefs, or any listed file has uncommitted changes (so user edits stay in separate commits).
- 2.3. Push restricted to those 2 files by 3 model-independent layers: permission rule allowing only `python3 .sync/sync.py`, explicit staging (never `git add -A`), and a marker-gated `.githooks/pre-commit` guard.
- 2.4. Scopes not hard-coded: no-arg → `universal/index_otg.md` + `universal/preferences.md`; `#sync <cp>` → `<cp>/CP_index_otg.md` + `<cp>/CP_instr.md`; file list read from the index, so cross-folder entries (e.g. `seek/context/*`) are handled and `seek/gcl/*` is never touched.
- 2.5. "Date Added" (Finder column) preserved on edits to `sessions/` + `seek/investigation/` via Pre/PostToolUse capture→restore hooks (`.sync/date_added.py`).
- 2.6. `#sync` is safe on Sonnet (use "Auto" mode, not Bypass) and even Haiku for the action itself —— safety is architectural; comms-protocol drift is the only downgrade risk.
- 2.7. §8.6 forbids the CC memory store; a one-off `override` was used to add the NO-CHAT-TEXT feedback memory.
