## 2. Decisions
- 2.1. **Root cause — `URL_RE` regex too strict.** The regex `re.compile(re.escape(PREFIX) + r"([0-9a-fA-F]{7,40}|main)/(\S+)")` required a SHA (7–40 hex chars) or the literal `main` between the repo slug and the file path. URLs that had neither (bare form: `…/dupbus-ceztuc-7cufVe/universal/writing_otg.md`) never matched, so `findall` returned nothing for them, `shamap` had no entry, and `repl` was never called — the script silently left those lines unchanged with no error.
- 2.2. **Fix — made the ref group optional.** Changed to `re.compile(re.escape(PREFIX) + r"(?:([0-9a-fA-F]{7,40}|main)/)?(\S+)")`. The `(?:…)?` wrapper makes the SHA/branch component optional. Bare URLs now match: group 1 (ref) = `None`, group 2 (path) = the file path (e.g. `universal/writing_otg.md`). In `repl`, `None != shamap[p]` is always `True`, so bare URLs are unconditionally counted as changed and correctly pinned. `sha_of()` works normally since those files ARE committed.
- 2.3. **No other code changed.** The `repl` function, `sha_of`, path extraction (`{p for _ref, p in …}`), dirty-check loop, and commit/push logic required no modification — the optional group integrates cleanly.
- 2.4. **Why 3 entries were bare.** `writing.md` (→ `writing_otg.md`), `replace.md`, and `debate.md` (→ `debate_otg.md`) were added to the index manually without running `#sync` afterwards, so they never got a SHA pinned. The old regex never caught them on subsequent `#sync` runs either, perpetuating the stale bare URL indefinitely.
