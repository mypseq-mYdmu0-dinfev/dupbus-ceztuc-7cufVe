## Wk1 (21/03/2026) —— Intro, Scope, Participant Access

### Research Output = Academic, Not Consultancy
- Enis explicitly: output is an academic dissertation, not a consulting report; industry experience can cause students to default to "consultancy writing style" — must resist; all opinions/arguments must be grounded in research, not practitioner experience alone
- ⚠️ Directly relevant: Culous' consulting bg = potential drift risk; must maintain academic register throughout A1–A3

### Research Proposal Structure (Enis' Intro)
- Intro + background + research title → aim → objectives → RQs (or hypotheses if quant) → lit review → methodology → Gantt → references
- RQs are provisional at proposal stage; refined post-lit-review
- Aim → objectives → lit review → RQs: this sequence is the "golden thread" and must be traceable throughout

### Methodology Principles
- Qual: feelings, perceptions, opinions, personal experiences → interviews or focus groups
- Quant: relationships between variables → questionnaire/survey
- Challenge type determines methodology, not the other way around
- Secondary data originally gathered from humans for another purpose = very complex ethics issue (requires re-consent from original participants); avoid unless non-human numerical data

## Wk3 (04/04/2026) —— Literature Review, Conceptual Framework, Multi-Org Confirmed

### Literature Review —— Structure and Purpose
- Lit review is NOT just reporting; must incorporate researcher's interpretive lens linked to aim/objectives
- Start broad (news, industry reports) → narrow via objective-linked keywords → develop into themed sections
- Each section structured around objectives (headings should not literally be the objective text — give them distinct labels)
- Lit review ends with a **conceptual framework or model** — visual synthesis of understanding

### Conceptual Framework —— Critical Rule for Qual
- Framework must emerge from the lit review (not predesigned upfront)
- **Qualitative: NO arrows** — arrows imply impact/causal measurement, which only quant can claim; use lines, shapes, thematic trees
- Quantitative: arrows between variables with hypotheses (H1, H2...) acceptable
- ⚠️ For our TPS project: framework will use non-directional connectors; updated version produced post-analysis in Ch5
- Enis shared his own qualitative PhD paper example: tree-shaped framework showing institutional partnership factors, KM activities, behavioural constructs — no arrows

### Framework-to-Analysis Loop (Full Logic Chain)
Lit review → conceptual framework → data collection instrument → findings → updated conceptual framework → discussion/analysis → conclusions/recommendations
- If data contradicts initial framework: update the framework; critically explain why; contradictory findings = equally valuable if discussed carefully
- "Finding that things don't work as expected is equally important, if not more important, if critically discussed"

### Qualitative Findings Presentation
- Present themes + 2–4 participant quotes per theme (in quotation marks, italics)
- After quote: brief interpretation linking it to the theme/concept
- "Interviewee 5, when asked about X, stated the following: [quote]" → explain what it means for the argument
- ⚠️ Do not present raw transcripts; quotes are evidence for thematic claims, not the claim itself

### NVivo and Coding Software
- NVivo = UoL-licensed; check student support for access
- Manual coding is equally valid; not penalised for not using software
- ⚠️ Only UoL-approved software permitted (not Atlas TI, Dedoose, or other third-party tools); confirm with student support or DA before using any platform

### Data Saturation —— Extended Elaboration (Wk6)
- Saturation = stop conducting interviews when nothing new is emerging; justified through evidence, not instinct
- Process: begin coding after interview 2–3; follow Braun and Clarke's six-phase thematic analysis; identify codes → primary/secondary codes → themes; saturation is demonstrable through coded data, not just a stated claim
- "Golden nuggets" — qualitative value is not only in pattern convergence but in divergences, contradictions, and interesting outliers across lived experiences; these feed the discussion and analysis chapters
- ⚠️ Disconfirming evidence coding (our standing rule from AD02.2.3/Concept Brief §8.1) is precisely this — actively look for what contradicts the dominant pattern

### Semi-Structured Interviews —— Consistency & Probing (Wk6)
- All participants receive the same core questions — deviating per person undermines reliability of the instrument
- Probing questions may differ between participants: if one respondent gives an exhaustive answer to Q3, probing Qs for Q3 may not be needed; those probes may surface later in Q8 or Q9 for the same participant, or arise in Q3 for another
- Structure remains constant; probing flexibility is the "semi" in semi-structured
- ⚠️ Our Guide A / Guide B / Guide C design is consistent with this — core question set fixed, probing questions adaptive

### Supervisor (DA) Process —— Detailed Confirmation (Wk6, Enis' own practice)
- DA reviews each draft chapter once → provides comments → student revises → DA does NOT re-review that chapter individually

- Students may share 1, 2, or 3 chapters at a time — DA's preference may vary
