## W3.1 — Literature Reviews

### Purpose (why lit review matters)
- Locates your research in existing knowledge; defines scope; demonstrates critical thinking
- Identifies gaps that justify your study
- ⭐ Refines aim, objectives, and RQs — RQs MUST emerge from lit review (not set in advance)
- Informs methodology by showing how others approached similar problems
- Provides conceptual architecture for data collection instrument

### Sources
- Academic journals (JSTOR, Web of Science, Scopus) = primary
- Books = useful starting point; can't be relied on exclusively for MBA project
- Grey literature (govt reports, industry reports) = acceptable supplement; validate carefully
- ❌ Wikipedia: never

### Evaluation criteria (Easterby-Smith et al. 2015)
- **Purpose** (why was this created?) / **Authorship** (who? authoritative?) / **Credibility** (peer-reviewed? method transparent?) / **Timeliness** (still valid for this topic?)

### Types of Lit Review
- ⭐ **Narrative review = most common for MBA final project** (broad overview, subjective interpretation)
- Systematic = rigorous inclusion/exclusion criteria; usually for A1 proposal scope
- Meta-analysis, scoping, rapid = less likely for MBA

### Three-Stage Framework (Smallbone & Quinton 2011)
1. **Search** — keyword strategy, Boolean logic, database selection; start broad, narrow down; keywords evolve
2. **Deconstruct** — assess each article: relevant? quality? method appropriate? findings consistent? conclusions match context?
3. **Reconstruct/Synthesise** — pull themes across multiple sources; do NOT summarise paper by paper; themes become sub-headings
- ⚠️ **Most common mistake:** summarising each paper individually instead of synthesising across them into themes

### Venn Diagram for Keywords
- Place topic in centre; place key concepts on outer circles; overlapping zones = combined search terms
- Use synonyms for each concept to widen search

### Conceptual Framework
- ⭐ **Lit review must end with a visual conceptual framework** (thematic map, model, or diagram)
- Shows your synthesis of key themes and their relationships
- Qual = no arrows (arrows imply causal impact); use lines, shapes, trees
- Quant = arrows + variables + hypotheses
- Framework updated post-analysis in Ch5 to reflect what data showed vs. what lit predicted
- ⭐ For TPS topic: framework should show relationships between TP dynamics, team cohesion, KM behaviour, and managerial responses — without causal arrows (qual)

### Literature Review Framework Table (useful tool)
- Columns: Source/Journal | Author | Year | Focus/Aims | Method | Country | Key relevant findings
- Build this as you read; use to organise reconstruction phase

## W3.2 — Reading Academic Papers

- Read abstract first → if relevant, read intro + conclusion → if still relevant, read selectively by section
- ⚠️ Don't stop at unfamiliar words — highlight and continue; context usually clarifies
- Good papers use section headings and strong opening sentences as navigation shortcuts
- Faster = more practice; don't try to read everything linearly from the start
- Look for: what prior research does this build on? aims? methods? key findings? how does it fit your topic?
