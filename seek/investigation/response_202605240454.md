Then report loop completion." → on SA's loop completion report, retire SA and spawn fresh SA to re-process the same job (Pre-Flight will recover via Tab 2 state)

**If OUTCOME = Skipped or Pending (SA completed loop):**
1. Check FLAGS for any ⚠️ requiring MA decision
2. If FLAGS contain escalation → assess; send one correction message; if SA still wrong on next report → retire and spawn fresh
3. If no issues → continue

**After any OUTCOME:**
1. Increment N; output to user: `🎯[N] **job(s) processed so far.**` (exact C2 format)
2. Re-read `cc_reminder.md` if N is a multiple of 5 (every 5 loops)
3. If no deterioration signals → SendMessage to SA: "Continue to next card."
4. If retiring SA → spawn fresh SA per § SA Spawn Instruction with correct [CARD_POSITION]
