## 3. Re 7 — Session Auto-Resume via Scheduled Task

**Answer to your question**: Yes. A ScheduleWakeup task persists independently of the current session's context state. If the session runs out of limit at 0330 and a task was scheduled to fire at 0400, it WILL fire at 0400 — starting a new Claude Code invocation with the scheduled prompt. So:

3.1. You write `202606010400` (or whatever the next reset time is) to `/seek/.claude/tmp/reset_time.md`
3.2. MA Session Start step 6.1 reads this file, calculates delay seconds, and schedules a `ScheduleWakeup(prompt="seek", ...)`
3.3. If limit runs out at any point before 0400: session ends, tasks and loop stop
3.4. At 0400: the scheduled task fires, new MA session starts, reads all mandatory files, runs Pre-Flight, resumes normally
3.5. After scheduling, MA clears `reset_time.md` (so it doesn't re-schedule on the resumed session's startup unless you set it again)

**Implementation**: Change 02 (step 6.1) is now live in main_ajap.md. The Bash command uses macOS `date -j -f` BSD syntax — verify it works on your machine before first use: `echo $(( $(TZ='Australia/Sydney' date -j -f "%Y%m%d%H%M" "202606010400" +%s) - $(date +%s) ))` should return a positive number of seconds.

**For 48hr continuous run**: set `reset_time.md` at the start. Each auto-resumed MA session picks up from Pre-Flight. No ⏳_ leftover problem (pre-flight CL check + pre-write approval handles it). The new MA session will also run the session start cleanup, reschedule its own heartbeat, etc.
