- 1.1. Durability rule: any fix that must survive compaction MUST live in a file MA re-reads on EVERY wake (`MA_hb.md`) or in the start/recovery gates (`seek/CLAUDE.md`, `main_ajap.md`). Chat/rlog-only fixes regressed (heartbeat, newtoyou, rapid-resume) —— see audit §2.4.
- 1.2. Watchdog form: SA2 = a second, independent, NEVER-churned persistent `Monitor` (deterministic bash), not an LLM agent (audit/response §3.2). Goal-identical, strictly more robust; file structure (`MA_hb.md` / `SA2_hb.md`) is the same either way.
- 1.3. Liveness fact (rlog 202606011118): `TaskList` does NOT enumerate background `Monitor` processes —— so MA must NOT rely on TaskList for Monitor liveness. The watchdog is the liveness mechanism.
