40. Re 27.2 — one loop per sub-agent invocation, main agent audits between loops:

This has an important implication: the Chat Rule (C1–C5 only) applies to the main agent's user-visible outputs, not to the sub-agent's internal reporting. The sub-agent can narrate freely ("Scored 75, Quick Apply, applying now...") — this is actually desirable, because the main agent uses that narration to audit without needing to inspect files.
- 40.2. **Can we SendMessage to the same sub-agent to continue it?** Yes. The Agent tool description explicitly states: "To continue a previously spawned agent, use SendMessage with the agent's ID or name as the `to` field — that resumes it with full context." So the model is: spawn once, SendMessage for each subsequent loop. The sub-agent retains its context from prior loops, which is useful (it knows which card it just processed, which tab is open, etc.). Retire only when deteriorated.
- 40.3. **What the main agent audits between loops:**
  - Sub-agent output was not verbose narration to the user (if it was, sub-agent leaked; retire)

  - Last AR CL does not contain `—`, `–`, or `+`

- 40.4. **Main agent's user-visible output between loops:** only S0.3 count (C2) and any C3/C4/C5 as needed. The main agent replaces the sub-agent as the source of the C2 count — sub-agent no longer needs to output S0.3 to the user, only to the main agent in its loop report.
